Step 1: Mastering AI Coding Agents

Effective Prompt Engineering

# Example: Android Framework Issue Analysis

CONTEXT:
- Android 14, Qualcomm SM7550 platform
- Module: SystemUI/NotificationService
- Build: eng-debug variant

ISSUE:
- ANR in NotificationManagerService
- Occurs when 50+ notifications pending
- Main thread blocked for 5+ seconds

LOGS:
[Attach full logcat with timestamps]
[Include system_server traces]
[Add dumpsys notification output]

TASK:
1. Identify root cause from stack trace
2. Propose fix with minimal side effects
3. Consider performance implications
4. Generate unit test for the fix
✅ Best Practice: Always provide complete context including platform, module, build variant, and full logs. The more specific your prompt, the better the AI output.

Step 3: Optimized Build Strategy

Conditional Compilation for Multiple Fixes

// Android.bp - Feature flag configuration
cc_defaults {
  name: "debug_features",
  cflags: [
    "-DFIX_ANR_ISSUE_1234",
    "-DFIX_MEMORY_LEAK_1235",
    "-DFIX_BOOT_CRASH_1236",
  ],
}

// SystemUI.cpp - Conditional fixes
#ifdef FIX_ANR_ISSUE_1234
  // AI-generated fix for ANR
  mNotificationQueue.setMaxSize(100);
  enableAsyncProcessing();
#endif

#ifdef FIX_MEMORY_LEAK_1235
  // AI-generated memory leak fix
  mBitmapCache.setRecyclePolicy(AGGRESSIVE);
#endif
✅ Best Practice: Use feature flags to enable/disable fixes during testing. This allows testing multiple solutions without rebuilding.
